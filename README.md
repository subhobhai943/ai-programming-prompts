## 💡 Usage Examples

### Example 1: Code Generation
```
You are an expert Python developer. I need you to create a utility function that calculates compound interest.

Requirements:
- Accept principal amount, annual interest rate, number of years, and compounding frequency
- Follow Python best practices
- Include proper error handling
- Add comprehensive comments explaining the logic
- Consider edge cases such as negative values, zero inputs, and very large numbers
- Optimize for readability and accuracy
```

### Example 2: Debugging Assistant
```
You are an expert debugger and JavaScript developer. I'm experiencing a runtime error with my code.

Problem Description:
- Expected behavior: Function should return filtered array
- Actual behavior: Getting "TypeError: Cannot read property 'filter' of undefined"
- Error message: TypeError: Cannot read property 'filter' of undefined
- Environment: Node.js 18.x, running in production

[Include your code here]
```

## 📖 Documentation

### Prompt Structure
Each prompt follows a consistent structure:
- **Title**: Clear description of the prompt's purpose
- **Description**: Detailed explanation of use cases
- **Prompt**: The actual prompt template with variables
- **Example**: Real-world usage example
- **Variables**: List of customizable parameters

### Variable Replacement
Replace placeholders like `{language}`, `{task_type}`, `{requirements}` with your specific values.

### Best Practices
- Always customize prompts for your specific context
- Include relevant technical details
- Specify your experience level
- Mention any constraints or limitations
- Ask for explanations along with code
